# user CLAUDE.md

ユーザから指示が与えられた場合にはまず、planモード・context7MCP・WebSearchなどを用いて、コードベースやライブラリの情報などを調査した上で、ユーザの要求に対する最適な実装方法を検討してください。
取りうる実装方法が複数ある場合には、ユーザにメリットとデメリットを提示し、最適な方法を選択してもらってください。

## Response Style

- 応答は日本語で行うこと
- 出力中で絵文字を絶対に使用しないこと

## Work Management

- Webフロントエンドの実装においてはどんな小さな実装でも`co-yuma`エージェントを**絶対に**使用すること
- サブエージェントにはユーザの指示を直接渡すのではなく、あなたが司令塔となり、サブエージェントには具体的なタスクを指示すること
- 渡されたタスクや必要な変更に並行して実行可能な箇所がある場合には**積極的に**、サブエージェントで並行して作業を進めること
- スクリプトの実行は絶対にバックエンド実行すること
- サブエージェントはバックグラウンド実行せず、必ずフォアグラウンドで実行すること

## Code Editing

- コードを編集する際、変更内容を説明するコメントを絶対に追加しない（JSDocなどのドキュメントコメントは除く）
- 必要な説明は会話の中で行う
- 与えたタスクに関連のない箇所の変更を行うことを禁止します。代わりにそれはユーザに報告してください(ex: 検索中に発見した別のバグやタイポなど)

## Command Line

- ブランチの切り替え/新規作成においては絶対に`git checkout`ではなく必ず`git switch`を使用すること
- ファイルのリネーム・移動には絶対に`git mv`を使用すること（git管理されているもののみ）
- 依存関係の変更は直接`package.json`や`package-lock.json`/`yarn.lock`を編集することを禁止します。必ずパッケージマネージャのコマンドを使用して行うこと
